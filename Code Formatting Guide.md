# Code Formatting Guide

Code auf Englisch, Ausgabe/UI auf deutsch

Beim Array die Klammern beim Datentypen (z.B. "String[] args")

vor und nach Klammern außer bei Funktionen immer Spaces (z.B. "if () {", "funktion()")

Strings immer mit Double Quotes (""String"")

chars immer mit Single Quotes ("'a'")

keine single Buchstaben für Variablen außer wenn nur innerhalb 2 Zeilen oder als index in Schleife. Parameter/Attribute sind voll ausgeschrieben
